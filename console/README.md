# AWS マネジメントコンソール操作の基本ハンズオン

AWS マネジメントコンソールの基本的な操作方法を学ぶハンズオンです。ReadOnly 権限のみで安全に実施できます。

## 前提条件

- AWS アカウントを持っていること
- ReadOnly アクセス権限（`ReadOnlyAccess` ポリシー）が付与された IAM ユーザーまたはロールでサインインすること

> ReadOnly 権限のため、リソースの作成・変更・削除はできません。閲覧操作のみを行います。

---

## 1. コンソールへのサインイン

### 1-1. サインイン

1. インストラクターのガイドに基づき AWS マネジメントコンソールにサインインする

### 1-2. コンソールのホーム画面を確認する

サインイン後、コンソールのホーム画面が表示されます。以下の要素を確認しましょう。

| 要素 | 場所 | 説明 |
|------|------|------|
| 検索バー | 画面上部中央 | サービス名やリソースを検索できる |
| リージョン選択 | 画面右上 | 現在のリージョンを確認・変更できる |
| アカウント情報 | 画面右上 | サインイン中のアカウント・ユーザー情報を確認できる |
| サービスメニュー | 画面左上「サービス」 | すべての AWS サービスをカテゴリ別に一覧できる |
| 最近アクセスしたサービス | ホーム画面中央 | 直近で使用したサービスへのショートカット |
| お気に入り | ナビゲーションバー | よく使うサービスをピン留めできる |

---

## 2. リージョンの確認と切り替え

### 2-1. 現在のリージョンを確認する

1. 画面右上のリージョン名（例：「東京」「アジアパシフィック (東京) ap-northeast-1」）を確認する

### 2-2. リージョンを切り替える

1. リージョン名をクリックする
2. ドロップダウンからリージョンの一覧が表示される
3. 「米国東部 (バージニア北部) us-east-1」を選択する
4. 画面右上のリージョン表示が切り替わったことを確認する
5. 同様の手順で「アジアパシフィック (東京) ap-northeast-1」に戻す

> リージョンによって利用可能なサービスや料金が異なります。通常、日本からの利用では東京リージョンを使用します。

---

## 3. サービスの検索とアクセス

### 3-1. 検索バーからサービスを探す

1. 画面上部の検索バーをクリックする（Windows の場合 ショートカット `Alt + S` でもよい）
2. 「s3」と入力する
3. 検索結果に「S3」サービスが表示される
4. クリックして S3 ダッシュボードを開く
5. ページ左上の AWS アイコンをクリックしてコンソールのホームページに戻る

### 3-2. お気に入りに追加する

1. 検索バーに「s3」と入力する
2. 検索結果の「S3」の左側にある☆アイコンをクリックする
3. ナビゲーションバーに S3 がピン留めされたことを確認する
4. 同様に「Bedrock」もお気に入りに追加してみる

---

## 4. AWS CloudShell を使ってみる

### 4-1. CloudShell を起動する

1. 画面下部のナビゲーションバーにある CloudShell アイコン（`>_` のようなアイコン）をクリックする
2. CloudShell のターミナルが画面下部に表示される（初回は起動に少し時間がかかります）

### 4-2. AWS CLI コマンドを実行する

CloudShell には AWS CLI がプリインストールされています。ReadOnly 権限で情報を取得してみましょう。

1. 現在のリージョンを確認する

```bash
echo $AWS_DEFAULT_REGION
```

> CloudShell では `aws configure` でリージョンが設定されていないため、環境変数 `AWS_DEFAULT_REGION` を参照します。

2. 現在のユーザー情報を確認する

```bash
aws sts get-caller-identity
```

3. Amazon Bedrock で使用できる基盤モデルの一覧を表示する
    - 実行後、表示を注視する場合は q キーを押下する

```bash
aws bedrock list-foundation-models
```


4. CloudShell の右上の X アイコンをクリックして　CloudShell を終了する

---

## 5. サインアウト

1. 画面右上のアカウント名（またはユーザー名）をクリックする
2. ドロップダウンメニューから「サインアウト」をクリックする
3. サインアウトが完了し、サインイン画面に戻ることを確認する
