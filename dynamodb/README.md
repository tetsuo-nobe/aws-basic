# Amazon DynamoDB ハンズオン

AWS マネジメントコンソールを使用して DynamoDB テーブルを作成し、アイテムの作成・編集・削除・検索を体験するハンズオンです。

## 前提条件

- AWS アカウントを持っていること
- 必要な IAM 権限が付与された IAM ユーザーでマネジメントコンソールにサインイン済みであること

---

## テーブル設計

このハンズオンでは、試験の点数を格納する `exam_score` テーブルを作成します。

### キー設計

| キー | 属性名 | 型 | 説明 |
|------|--------|-----|------|
| パーティションキー | `student_id` | 文字列 (S) | 受験者の ID |
| ソートキー | `exam_id` | 文字列 (S) | 試験の ID |

### サンプルデータ

DynamoDB はスキーマレスなデータベースのため、アイテムごとに異なる属性を持つことができます。このハンズオンでは、文系・理系の受験者で異なる科目のスコアを格納します。

| student_id | exam_id | student_name | course | japanese | english | japanese_history | world_history | math | physics |
|-----------|---------|-------------|--------|---------|---------|-----------------|--------------|------|---------|
| STU001 | EXAM2025A | 受験者A | 文系 | 78 | 82 | 71 | - | - | - |
| STU002 | EXAM2025A | 受験者B | 文系 | 85 | 90 | - | 88 | - | - |
| STU003 | EXAM2025A | 受験者C | 理系 | - | 75 | - | - | 92 | 88 |

> DynamoDB ではアイテムごとに属性が異なっても問題ありません。これがリレーショナルデータベースとの大きな違いの 1 つです。上の表で「-」となっている属性は、そのアイテムには存在しません。

---

## 1. テーブルの作成

### 1-1. DynamoDB コンソールを開く

1. AWS マネジメントコンソールにサインインする
2. 上部の検索バーに「DynamoDB」と入力し、DynamoDB サービスを選択する

### 1-2. テーブルを作成する

1. 左メニューから「テーブル」をクリックする
2. 「テーブルの作成」ボタンをクリックする
3. 以下の項目を設定する

| 項目 | 設定値 |
|------|--------|
| テーブル名 | `exam_score` |
| パーティションキー | `student_id`（文字列） |
| ソートキー | `exam_id`（文字列） |

4. 「テーブル設定」は「デフォルト設定」のままにする
5. 「テーブルの作成」をクリックする
6. ステータスが「アクティブ」になるまで待つ（数秒〜1 分程度）

---

## 2. アイテムの作成

### 2-1. 受験者 A のアイテムを作成する（文系：国語・英語・日本史）

1. 作成した `exam_score` テーブルをクリックする
2. 「テーブルアイテムの探索」をクリックする
3. 「アイテムを作成」ボタンをクリックする
4. 入力形式を右上の「JSON ビュー」に切り替える
5. 以下の JSON を入力する

```json
{
  "student_id": {"S": "STU001"},
  "exam_id": {"S": "EXAM2025A"},
  "student_name": {"S": "受験者A"},
  "course": {"S": "文系"},
  "japanese": {"N": "78"},
  "english": {"N": "82"},
  "japanese_history": {"N": "71"}
}
```

6. 「アイテムを作成」をクリックする

### 2-2. 受験者 B のアイテムを作成する（文系：国語・英語・世界史）

1. 「アイテムを作成」ボタンをクリックする
2. 「JSON ビュー」に切り替え、以下の JSON を入力する

```json
{
  "student_id": {"S": "STU002"},
  "exam_id": {"S": "EXAM2025A"},
  "student_name": {"S": "受験者B"},
  "course": {"S": "文系"},
  "japanese": {"N": "85"},
  "english": {"N": "90"},
  "world_history": {"N": "88"}
}
```

3. 「アイテムを作成」をクリックする

### 2-3. 受験者 C のアイテムを作成する（理系：英語・数学・物理）

1. 「アイテムを作成」ボタンをクリックする
2. 「JSON ビュー」に切り替え、以下の JSON を入力する

```json
{
  "student_id": {"S": "STU003"},
  "exam_id": {"S": "EXAM2025A"},
  "student_name": {"S": "受験者C"},
  "course": {"S": "理系"},
  "english": {"N": "75"},
  "math": {"N": "92"},
  "physics": {"N": "88"}
}
```

3. 「アイテムを作成」をクリックする

### 2-4. アイテムの確認

1. 「テーブルアイテムの探索」画面に 3 件のアイテムが表示されていることを確認する
2. 各アイテムの属性が異なっていることを確認する（受験者ごとに科目が違う）

> 受験者 A には `japanese_history` があり、受験者 B には `world_history` があり、受験者 C には `math` と `physics` があります。このようにアイテムごとに異なる属性を持てるのが DynamoDB の特徴です。

---

## 3. アイテムの編集

### 3-1. 受験者 A の英語のスコアを修正する

受験者 A の英語のスコアを 82 から 85 に修正します。

1. 「テーブルアイテムの探索」画面で `STU001` のアイテムをクリックする
2. `english` 属性の値 `82` を `85` に変更する
3. 「変更を保存」をクリックする

### 3-2. 受験者 C に属性を追加する

受験者 C に `chemistry`（化学）のスコアを追加します。

1. `STU003` のアイテムをクリックする
2. 「新しい属性の追加」をクリックし、「数値」を選択する
3. 属性名に `chemistry`、値に `80` を入力する
4. 「変更を保存」をクリックする

> DynamoDB ではテーブルのスキーマを変更することなく、既存のアイテムに新しい属性を自由に追加できます。

---

## 4. アイテムの検索

### 4-1. Query（クエリ）で特定の受験者を検索する

パーティションキーを指定して、特定の受験者のアイテムを検索します。

1. 「テーブルアイテムの探索」画面で、検索方法が「クエリ」になっていることを確認する（「スキャン」になっている場合は「クエリ」に切り替える）
2. 以下の条件を設定する

| 項目 | 設定値 |
|------|--------|
| student_id（パーティションキー） | `STU001` |

3. 「実行」ボタンをクリックする
4. 受験者 A のアイテムのみが表示されることを確認する

### 4-2. Query でパーティションキーとソートキーを組み合わせて検索する

1. 以下の条件を設定する

| 項目 | 設定値 |
|------|--------|
| student_id（パーティションキー） | `STU002` |
| exam_id（ソートキー） | `EXAM2025A` |

2. 「実行」ボタンをクリックする
3. 受験者 B の EXAM2025A のアイテムが表示されることを確認する

### 4-3. Scan（スキャン）で全件取得する

1. 検索方法を「スキャン」に切り替える
2. フィルター条件を設定せずに「実行」ボタンをクリックする
3. テーブル内のすべてのアイテム（3 件）が表示されることを確認する

> Scan はテーブル全体を読み取るため、データ量が多い場合はコストとパフォーマンスに影響します。本番環境では可能な限り Query を使用することが推奨されます。

### 4-4. Scan にフィルター条件を追加する

文系の受験者のみを検索してみましょう。

1. 検索方法が「スキャン」になっていることを確認する
2. 「フィルター」セクションで以下を設定する

| 項目 | 設定値 |
|------|--------|
| 属性名 | `course` |
| 型 | 文字列 |
| 条件 | 次と等しい |
| 値 | `文系` |

3. 「実行」ボタンをクリックする
4. 受験者 A と受験者 B の 2 件のみが表示されることを確認する

> フィルター条件は Scan で読み取った後にサーバー側で適用されます。読み取り自体はテーブル全体に対して行われるため、フィルターを使っても読み取りコストは変わりません。

---

## 5. アイテムの削除

### 5-1. 受験者 C のアイテムを削除する

1. 「テーブルアイテムの探索」画面で `STU003` のアイテムのチェックボックスを選択する
2. 「アクション」ドロップダウンから「アイテムの削除」を選択する
3. 確認ダイアログで「削除」をクリックする
4. アイテムが削除され、2 件になっていることを確認する

---

## 6. クリーンアップ

ハンズオン終了後、作成したテーブルを削除します。

### 6-1. テーブルを削除する

1. 左メニューから「テーブル」をクリックする
2. `exam_score` テーブルのラジオボタンを選択する
3. 「削除」ボタンをクリックする
4. 確認画面で「すべての CloudWatch アラームを削除」にチェックが入っていることを確認する
5. 確認のテキストフィールドに `確認` と入力する
6. 「テーブルの削除」をクリックする
7. テーブルが削除されることを確認する

---

## まとめ

このハンズオンで学んだことを振り返りましょう。

| 学んだこと | 内容 |
|-----------|------|
| テーブル作成 | パーティションキーとソートキーを指定してテーブルを作成する |
| スキーマレス | アイテムごとに異なる属性を持つことができる |
| アイテム作成 | JSON 形式でアイテムを作成する |
| アイテム編集 | 既存の属性値の変更や新しい属性の追加ができる |
| Query | パーティションキー（とソートキー）を指定した効率的な検索 |
| Scan | テーブル全体を読み取る検索（フィルター条件の追加も可能） |
| Query と Scan の違い | Query はキーを指定した効率的な検索、Scan は全件読み取りのためコストが高い |
| アイテム削除 | 不要なアイテムを削除する |
