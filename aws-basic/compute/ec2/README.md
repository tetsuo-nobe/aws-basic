# EC2 インスタンス作成ハンズオン

AWS マネジメントコンソールを使用して EC2 インスタンスを作成し、Session Manager で接続するまでの手順です。

## 前提条件

- AWS アカウントを持っていること
- IAM ロール `my-role-for-session-manager` が作成済みであること
  - `AmazonSSMManagedInstanceCore` ポリシーがアタッチされていること
- デフォルト VPC が存在すること

---

## 1. EC2 インスタンスの作成

### 1-1. EC2 ダッシュボードを開く

1. AWS マネジメントコンソールにサインインする
1. 上部の検索バーに「EC2」と入力し、EC2 サービスを選択する
1. 左メニューから「インスタンス」をクリックする
1. 「インスタンスを起動」ボタンをクリックする

### 1-2. 名前とタグ

1. 「名前」に任意のインスタンス名を入力する（例：`my-handson-instance`）

### 1-3. AMI の選択

1. 「Amazon Linux 2023 AMI」を選択する（デフォルトで選択済み）
1. アーキテクチャは「64 ビット (x86)」のままにする

### 1-4. インスタンスタイプの選択

1. インスタンスタイプのドロップダウンから `t3.micro` を選択する

### 1-5. キーペア（ログイン）

1. 「キーペア名」のドロップダウンから「キーペアなしで続行」を選択する

> SSH ではなく Session Manager で接続するため、キーペアは不要です。

### 1-6. ネットワーク設定

1. 「からの SSH トラフィックを許可」の **チェックを解除** する
1. 「インターネットからの HTTP トラフィックを許可」を **チェックする**

> Session Manager は SSM エージェントが HTTPS (443) でアウトバウンド通信を行うため、インバウンドルールは不要です。

### 1-7. ストレージの設定

1. デフォルトのまま変更しない（8 GiB gp3）

### 1-8. 高度な詳細

1. 「高度な詳細」セクションを展開する
1. 「IAM インスタンスプロファイル」のドロップダウンから `my-role-for-session-manager` を選択する

### 1-9. インスタンスの起動

1. ページ右下の「インスタンスを起動」ボタンをクリックする
1. 「すべてのインスタンスを表示」をクリックする
1. 「ステータスチェック」が「3/3 のチェックに合格しました」になるまで待つ（3〜4 分程度）

---

## 2. Session Manager で接続する

### 2-1. EC2 コンソールから接続

1. EC2 ダッシュボードの「インスタンス」画面で、作成したインスタンスを選択する
1. 「接続」ボタンをクリックする
1. 「SSM Session Manager」タブを選択する
1. 「接続」ボタンをクリックする

ブラウザ上でターミナルセッションが開き、インスタンスに接続できます。

* 次のコマンドを実行してみる
    - 現在の OS ユーザー名表示: `whoami`
    - カレントディレクトリ表示: `pwd`
    - OS ユーザーのホームディレクトリ表示: `echo $HOME`
    - 他の OS ユーザー: ec2-user にスイッチ: `sudo su - ec2-user`


---

## 3. Apache HTTP Server のインストール

Session Manager で接続した状態で、以下のコマンドを順に実行します。

1. パッケージを最新に更新する

```bash
sudo dnf update -y
```

2. Apache HTTP Server をインストールする

```bash
sudo dnf install -y httpd
```

3. Apache を起動し、OS 起動時に自動起動するよう設定する

```bash
sudo systemctl start httpd
sudo systemctl enable httpd
```

4. Apache が起動していることを確認する

```bash
sudo systemctl status httpd
```

> `Active: active (running)` と表示されれば OK です。

---

## 4. HTML ファイルの作成

1. `Hello EC2!` を表示する HTML ファイルを作成する

```bash
echo '<h1>Hello EC2!</h1>' | sudo tee /var/www/html/index.html
```

1. ローカルで動作確認する

```bash
curl http://localhost
```

> `<h1>Hello EC2!</h1>` と表示されれば成功です。

#### ブラウザからアクセスする場合

1. EC2 ダッシュボードでインスタンスを選択する。
1. インスタンス詳細画面で パブリック IPv4 を確認する。
1. ブラウザで `http://<インスタンスのパブリック IPv4>` にアクセスする。


---

## 5. クリーンアップ

ハンズオン終了後、不要な課金を防ぐためにインスタンスを終了します。

1. Session Manager のブラウザのラブを閉じる
1. EC2 ダッシュボードの「インスタンス」画面で、作成したインスタンスを選択する
2. 「インスタンスの状態」ドロップダウンから「インスタンスを終了 (削除)」を選択する
3. 確認ダイアログで「終了 (削除)」をクリックする
