# AWS Lambda 関数作成ハンズオン

AWS マネジメントコンソールを使用して Lambda 関数を作成し、API Gateway 経由で Web ブラウザからアクセスするまでの手順です。

## 前提条件

- AWS アカウントを持っていること

---

## 1. Lambda 関数の作成

### 1-1. Lambda コンソールを開く

1. AWS マネジメントコンソールにサインインする
2. 上部の検索バーに「Lambda」と入力し、Lambda サービスを選択する
3. 「関数を作成」ボタンをクリックする

### 1-2. 関数の基本設定

1. 「一から作成」を選択する
2. 以下の項目を設定する

| 項目 | 設定値 |
|------|--------|
| 関数名 | 任意の名前（例：`my-handson-function`） |
| ランタイム | Python 3.14 |
| アーキテクチャ | x86_64 |

### 1-3. 実行ロールの設定

1. 「デフォルトの実行ロールの変更」セクションを展開する
2. 「デフォルトロールを作成」を選択する（デフォルト）

### 1-4. 関数の作成

1. 「関数を作成」ボタンをクリックする
2. 関数が作成され、コードエディタ画面が表示される

---

## 2. コードの編集

### 2-1. コードを置き換える

1. 「コード」タブが選択されていることを確認する
2. コードエディタに表示されている `lambda_function.py` の内容をすべて削除する
3. 以下のコードを貼り付ける

```python
import json

# 6行目の blue を green か red か black に変えてみよう
# 7行目の xxx を自分の名前(ローマ字）に変えてみよう
html_response = "<HTML>" \
 + "<BODY bgcolor='blue'>" \
 + "<H2><font color='white'>Hello! xxx</font></H2>" \
 + "</BODY></HTML>"

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': html_response,
        'headers': {
                    'Content-Type': 'text/html; charset=utf-8'
                    
                }
    }
```

4. 必要に応じてコードをカスタマイズする
   - 6行目の `blue` を `green`、`red`、`black` などに変更して背景色を変える
   - 7行目の `xxx` を自分の名前（ローマ字）に変更する

### 2-2. コードをデプロイする

1. 「Deploy」ボタンをクリックする
2. 「関数 ○○ が正常に更新されました。」と表示されることを確認する

### 2-3. テスト実行

1. 「テスト」タブをクリックする
1. 「テスト」ボタンをクリックする
1. 「実行中の関数: 成功」と表示されることを確認する
1. 「詳細」を展開表示する
1. 実行結果が表示され、`statusCode: 200` と HTML が表示されることを確認する

---

## 3. API Gateway との統合

### 3-1. トリガーの追加

1. Lambda 関数の画面上部にある「関数の概要」セクションで「トリガーを追加」をクリックする
2. トリガーのドロップダウンから「API Gateway」を選択する
3. 以下の項目を設定する

| 項目 | 設定値 |
|------|--------|
| Intent | Create a new API |
| API タイプ | REST API |
| セキュリティ | オープン |

4. 「追加」ボタンをクリックする

### 3-2. API エンドポイントの確認

1. 「トリガー」セクションに「API エンドポイント」の URL が表示される
1. この URL をコピーする

### 3-3. ブラウザからアクセスする

1. コピーした API エンドポイントの URL をブラウザのアドレスバーに貼り付けてアクセスする
2. 背景色付きのページに「Hello! xxx」（または変更した名前）が表示されれば成功

---

## 4. クリーンアップ

ハンズオン終了後、不要なリソースを削除します。

### 4-1. Lambda 関数の削除

1. Lambda コンソールで作成した関数を選択する
2. 「アクション」ドロップダウンから「関数の削除」を選択する
3. 確認ダイアログで `確認` を入力して「削除」をクリックする

### 4-2. API Gateway の削除

1. 上部の検索バーに「API Gateway」と入力し、API Gateway サービスを選択する
1. 作成された API (例: my-handson-function-API) のラジオボタンを選択する
1. 「削除」を選択する
1. 確認ダイアログで `確認` を入力して「削除」をクリックする

### 4-3. IAM ロールの削除

1. 上部の検索バーに「IAM」と入力し、IAM サービスを選択する
2. 左メニューから「ロール」をクリックする
3. Lambda 関数作成時に自動生成されたロール（例：`my-handson-function-role-xxxxxxxx`）を検索して選択する
4. 「削除」ボタンをクリックする
5. 確認ダイアログでロール名を入力し「削除」をクリックする
