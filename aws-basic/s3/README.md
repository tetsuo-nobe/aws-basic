# Amazon S3 ハンズオン

S3 バケットの作成、オブジェクトのアップロード、署名付き URL によるアクセス、パブリックアクセスの設定までを体験するハンズオンです。

## 前提条件

- 必要な IAM ポリシーが付与された IAM ユーザーでマネジメントコンソールにサインイン済みであること
- インストラクターから提供されたファイル（`index.html` を含む）が手元にあること

---

## 1. バケットの作成

### 1-1. S3 コンソールを開く

1. 検索バーに「s3」と入力し、S3 サービスを選択する

### 1-2. バケットを作成する

1. 「バケットを作成」ボタンをクリックする
2. 以下の項目を設定する

| 項目 | 設定値 |
|------|--------|
| バケット名 | `tnobect0000-` で始まる任意の名前（例：`tnobect0000-12345`） |

3. その他の設定はデフォルトのまま「バケットを作成」をクリックする
4. バケットが正常に作成されることを確認する

> バケット名はグローバルで一意である必要があります。既に使われている名前の場合はエラーになるため、別の名前を試してください。

---

## 2. ファイルのアップロード

### 2-1. オブジェクトをアップロードする

* バケット作成完了後の続きとして

1. 「アップロード」ボタンをクリックする
1. 「ファイルを追加」をクリックし、インストラクターから提供された index.html ファイルを選択する
1. 「アップロード」ボタンをクリックする
1. アップロードが成功することを確認する
1. 「閉じる」をクリックしてバケットの画面に戻る

---

## 3. オブジェクト URL でのアクセス確認（アクセス不可）

### 3-1. index.html のオブジェクト URL を確認する

1. アップロードしたファイル一覧から `index.html` をクリックする
2. 「プロパティ」タブを開く
3. 「オブジェクト URL」に表示されている URL をコピーする

> オブジェクト URL の形式：`https://<バケット名>.s3.ap-northeast-1.amazonaws.com/index.html`

### 3-2. ブラウザからアクセスする（失敗を確認）

1. ブラウザの新しいタブを開く
2. コピーしたオブジェクト URL を貼り付けてアクセスする
3. **AccessDenied（アクセス拒否）のエラーが表示される**ことを確認する

> S3 バケットはデフォルトで非公開です。オブジェクト URL に直接アクセスしても、認証なしではアクセスできません。

---

## 4. 署名付き URL でのアクセス確認（アクセス可能）

### 4-1. 署名付き URL を生成する

1. S3 コンソールで `index.html` のページに戻る
2. 「オブジェクトアクション」ドロップダウンをクリックする
3. 「署名付き URL で共有」を選択する
4. 有効期限（分）を `1` に設定する
5. 「署名付き URL を作成」をクリックする
6. 署名付き URL がクリップボードにコピーされた状態になる

### 4-2. ブラウザからアクセスする（成功を確認）

1. ブラウザの新しいタブを開く
2. コピーした署名付き URL を貼り付けてアクセスする
3. **index.html の内容が正常に表示される**ことを確認する

> 署名付き URL には認証情報と有効期限が含まれているため、一時的にオブジェクトへのアクセスが可能になります。有効期限が切れるとアクセスできなくなります。

---

## 5. パブリックアクセスの設定

### 5-1. ブロックパブリックアクセスを解除する

1. S3 コンソールでバケット一覧に戻り、作成したバケットをクリックする
2. 「アクセス許可」タブをクリックする
3. 「ブロックパブリックアクセス (バケット設定)」セクションの「編集」をクリックする
4. 「パブリックアクセスをすべてブロック」の**チェックを外す**
5. 「変更の保存」をクリックする
6. 確認画面で「確認」と入力し、「確認」ボタンをクリックする

### 5-2. バケットポリシーを設定する

1. 同じ「アクセス許可」タブの「バケットポリシー」セクションで「編集」をクリックする
2. 以下の JSON を貼り付ける（`tnobect0000-12345` は自分のバケット名に置き換える）

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::tnobect0000-12345/*"
    }
  ]
}
```

> 例：バケット名が `tnobect0000-s3-handson` の場合、Resource は `arn:aws:s3:::tnobect0000-s3-handson/*` になります。

3. 「変更の保存」をクリックする

*  これでバケットは Public アクセス可能となりました。

---

## 6. オブジェクト URL でのアクセス確認（アクセス可能）

### 6-1. オブジェクト URL で再度アクセスする

1. バケット内の `index.html` をクリックする
2. 「プロパティ」タブの「オブジェクト URL」をコピーする
3. ブラウザの新しいタブを開く
4. コピーしたオブジェクト URL を貼り付けてアクセスする
5. **index.html の内容が正常に表示される**ことを確認する

> バケットポリシーで `s3:GetObject` を全員（`Principal: "*"`）に許可したため、署名なしの URL でもアクセスできるようになりました。

---

## 7. クリーンアップ

ハンズオン終了後、作成したリソースを削除します。

### 7-3. オブジェクトを削除する

1. バケット内のオブジェクトをすべて選択する
2. 「削除」ボタンをクリックする
3. 確認画面で `完全に削除` と入力し、「オブジェクトの削除」をクリックする
4. 「閉じる」をクリックする

### 7-4. バケットを削除する

1. S3 のバケット一覧画面に戻る
2. 作成したバケットのラジオボタンを選択する
3. 「削除」ボタンをクリックする
4. 確認画面でバケット名を入力し、「バケットを削除」をクリックする

---

## まとめ

このハンズオンで学んだことを振り返りましょう。

| 学んだこと | 内容 |
|-----------|------|
| バケット作成 | 命名規則に従ったバケットの作成 |
| オブジェクトのアップロード | コンソールからのファイルアップロード |
| オブジェクト URL | デフォルトでは非公開のためアクセス不可 |
| 署名付き URL | 一時的な認証付きアクセスの提供 |
| ブロックパブリックアクセス | バケット単位でのパブリックアクセス制御 |
| バケットポリシー | JSON ポリシーによるアクセス許可の設定 |
| パブリックアクセス | ポリシー設定後は署名なし URL でアクセス可能 |
